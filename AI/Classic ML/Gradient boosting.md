# Градиентный бустинг (GBDT)

**XGBoost, LightGBM, CatBoost.** Три эти библиотеки — современный стандарт для табличных данных. Они основаны на градиентном бустинге (GBDT), но реализуют его по-разному.

Все три алгоритма строят ансамбль деревьев последовательно, где каждое следующее дерево исправляет ошибки предыдущего. Но в деталях реализации есть ключевые различия.

```
Общий принцип: строим деревья последовательно, каждое исправляет ошибки предыдущих

F₀(x) = среднее значение
F₁(x) = F₀(x) + learning_rate × tree₁(x)
F₂(x) = F₁(x) + learning_rate × tree₂(x)
...
Fₘ(x) = сумма всех деревьев
```

# XGBoost (eXtreme Gradient Boosting): надёжный

* Главная идея - Использование второго порядка (гессиан) при оптимизации — более точные обновления.
* Рост деревьев - Глубинный (level-wise) — сбалансированные деревья.
* Регуляризация - L1 и L2 встроены в функцию потерь.
* Обработка пропусков - Автоматическая (оптимальное направление).

## Ключевые фичи:

| Фича | Что делает
|--|--
| Histogram-based splits | Бины вместо перебора всех значений — быстрее
| L1/L2 регуляризация | `reg_alpha`, `reg_lambda` — контроль сложности
| Column sampling | Случайная подвыборка признаков — меньше переобучение
| Missing values | Автоматическое направление в левый/правый узел
| Early stopping | Остановка, когда validation не улучшается
| GPU support | `tree_method='gpu_hist'`

## Сильные стороны

* Золотой стандарт индустрии, самый популярный
* Стабильность и предсказуемость результатов
* Отличная документация и комьюнити
* Поддержка всех языков (Python, R, Java, C++, Julia)
* GPU acceleration (но медленнее, чем у конкурентов)

## Как работает

* Pre-sorted algorithm (по умолчанию): сортирует признаки перед поиском лучшего разбиения
* Approximate algorithm: гистограммный метод для больших данных
* Sparsity-aware: автоматически обрабатывает пропуски
* Weighted quantile sketch: эффективный поиск порогов

## Ключевые параметры

```py
import xgboost as xgb

model = xgb.XGBClassifier(
    # Основные
    n_estimators=1000,           # Количество деревьев
    learning_rate=0.05,          # Темп обучения (eta)
    max_depth=6,                 # Глубина дерева
    
    # Регуляризация (важно)
    reg_alpha=0,                 # L1 (Lasso)
    reg_lambda=1,                # L2 (Ridge)
    min_child_weight=1,          # Минимальная сумма весов в листе
    
    # Subsampling
    subsample=0.8,               # Доля объектов для каждого дерева
    colsample_bytree=0.8,        # Доля признаков для каждого дерева
    colsample_bylevel=0.8,       # Доля признаков для каждого уровня
    
    # Производительность
    tree_method='hist',          # 'exact', 'hist', 'gpu_hist'
    n_jobs=-1,                   # Параллелизация
    
    # Early stopping
    eval_metric='auc',           # Метрика для валидации
    early_stopping_rounds=50,    # Остановка если нет улучшений
)
```

**Сильные стороны:**
* Хорошая документация и огромное сообщество
* Нужна максимальная стабильность и воспроизводимость
* Умеренная настройка гиперпараметров — прощает ошибки
* Отличная интеграция с sklearn (Pipeline, GridSearch)
* Работаете с не очень большими данными (< 1M строк)
* Нужно GPU ускорение (но есть нюансы)

**Слабые стороны:**
* Медленнее LightGBM на больших данных
* Требует ручного кодирования категорий (One-Hot, Label Encoding)
* Потребление памяти выше, чем у LightGBM (pre-sorted алгоритм)

# LightGBM (Light Gradient Boosting Machine): быстрый и эффективности

* Главная инновация - Leaf-wise рост (всегда сплитуем лист с макс. gain) + гистограммный алгоритм
* Ускорение - GOSS (градиентная выборка) + EFB (связывание редких признаков)
* Память - Самая экономичная из трёх
* Категории - Поддерживает, но требует int-кодирования

## Ключевые фичи

| Фича | Что делает
|--|--
| Leaf-wise growth | Растёт по листьям, а не по уровням — меньше узлов, точнее
| Gradient-based One-Side Sampling (GOSS) | Сэмплирует только важные градиенты — быстрее
| Exclusive Feature Bundling (EFB) | Группирует разреженные признаки — меньше памяти
| Native categorical features | Поддержка category dtype (но не так хорошо как CatBoost)
| Extremely fast | В 5-10x быстрее XGBoost на больших данных

## Сильные стороны

* Скорость: в 10-100 раз быстрее XGBoost на больших данных
* Эффективность памяти: гистограммный подход по умолчанию
* Отлично работает на огромных датасетах (миллионы строк)
* Native support categorical features (без One-Hot!)
* Лучшая масштабируемость
* Отличная поддержка GPU
* Leaf-wise даёт лучшую точность при правильной настройке

## Как работает 

* Histogram-based: дискретизация признаков в бины (быстрее + меньше памяти)
* GOSS (Gradient-based One-Side Sampling):
    - Сохраняет объекты с большим градиентом
    - Случайно сэмплирует объекты с маленьким градиентом
    - Ускоряет обучение без потери точности
* EFB (Exclusive Feature Bundling): объединяет разреженные признаки
* Leaf-wise рост: растит дерево в глубину (лучше точность, но риск переобучения)

## Минусы:

* Склонен к переобучению — требует аккуратной настройки `num_leaves` и `min_data_in_leaf`
* Параметров много — сложнее стартовать новичкам
* Менее стабильный — чувствителен к гиперпараметрам
* Медленнее CatBoost на инференсе (по некоторым данным)

## Ключевые параметры

```py
import lightgbm as lgb

model = lgb.LGBMClassifier(
    # Основные
    n_estimators=1000,
    learning_rate=0.05,
    max_depth=-1,                # -1 = no limit (лучше контролировать через num_leaves) ограничение глубины (борьба с переобучением)
    num_leaves=31,               # Главный параметр сложности (вместо max_depth, < 2^max_depth (типично 31-255)
    
    # GOSS и сэмплирование
    max_bin=255,                 # Количество бинов для гистограммы
    bagging_fraction=0.8,        # Доля объектов (subsample)
    bagging_freq=5,              # Как часто делать bagging (каждые 5 итераций)
    feature_fraction=0.8,        # Доля признаков
    
    # Регуляризация
    lambda_l1=0,                 # L1
    lambda_l2=1,                 # L2
    min_child_samples=20,        # Минимум объектов в листе (выше → менее переобучение)
    min_child_weight=0.001,      # Минимальная сумма градиентов
    
    # Категориальные признаки (ВАЖНО!)
    categorical_feature='auto',  # 'auto' или список индексов
    
    # Производительность
    device='cpu',                # 'cpu' или 'gpu'
    n_jobs=-1,
    
    # Early stopping
    eval_metric='auc',
)
```

### Когда использовать
* Большие данные (> 100K строк, оптимален для миллионов)
* Нужна максимальная скорость обучения
* Много категориальных признаков (без One-Hot)
* Online обучение или частые переобучения
* Ограниченная память
* Позже можно потюнить параметры для максимального качества

**Минусы:**
* Leaf-wise рост может переобучаться на маленьких данных
* Меньше стабильности на малых выборках
* Сложнее тюнить (больше параметров)


# CatBoost — эксперт по категориям

* Главная инновация - Ordered Boosting (борьба с target leakage) + симметричные деревья
* Категории - Лучшая в мире поддержка: строки без предобработки
* Инференс - Самый быстрый (благодаря симметричным деревьям)
* Переобучение - Самая устойчивая из трёх

## Ключевые фичи

| Фича | Что делает
|--|--
| Ordered Target Statistics | Обработка категорий без one-hot — устойчиво к переобучению
| Ordered boosting | Устраняет смещение предсказаний (bias)
| Symmetric trees | Все узлы одного уровня используют один сплит — регуляризация
| Автоматическая обработка пропусков | Как категорию или направление в узле
| Меньше гиперпараметров | Работает "из коробки" лучше

## Сильные стороны

* Лучшая работа с категориями "из коробки" (без предобработки)
* Ordered Boosting: борьба с target leakage
* Автоматическая обработка пропусков и текстов
* Отличная интерпретируемость (встроенные SHAP)
* Минимум гиперпараметров для тюнинга

## Как работает

* Ordered Target Statistics:
    - Кодирует категории через среднее таргета
    - Использует только "прошлые" объекты (временной порядок)
    - Избегает утечки данных
* Symmetric Trees: все деревья одинаковой структуры (быстрее инференс)
* Perfect Handling of Categorical Features:
    - Автоматические комбинации категорий
    - Не нужно делать One-Hot или Label Encoding
* Native support for text features: TF-IDF + обработка текста

## Ключевые параметры

```py
from catboost import CatBoostClassifier

model = CatBoostClassifier(
    # Основные
    iterations=1000,             # Количество деревьев (n_estimators)
    learning_rate=0.05,
    depth=6,                     # Глубина дерева (max_depth)(симметричных деревьев)
    
    # Регуляризация
    l2_leaf_reg=3,               # L2 регуляризация (аналог lambda)
    min_data_in_leaf=1,          # Минимум объектов в листе
    
    # Категориальные признаки (ГЛАВНАЯ ФИШКА!)
    cat_features=['city', 'education', 'marital_status'],  # Имена или индексы
    one_hot_max_size=2,          # One-Hot для категорий с <= N уникальных значений
    
    # Производительность
    task_type='CPU',             # 'CPU' или 'GPU'
    thread_count=-1,
    
    # Loss function
    loss_function='Logloss',     # 'Logloss', 'CrossEntropy', 'AUC'
    
    # Early stopping
    eval_metric='AUC',
    early_stopping_rounds=50,
    
    # Валидация
    use_best_model=True,         # Использовать лучшую итерацию
)
```

## Когда использовать:

* Много категориальных признаков (особенно high-cardinality: user_id, geo)
* Нужно быстро получить качественную модель без тонкой настройки
* Важна скорость инференса (real-time, edge devices)
* Автоматизация ML пайплайнов (меньше предобработки)
* Нужна интерпретация (SHAP, feature importance)
* В данных есть дрейф (Ordered Boosting устойчивее)
* Есть текстовые признаки
* Маленькие данные (лучше обобщение)

**Минусы:**
* Медленнее обучение (особенно на больших данных)
* Больше потребление памяти
* Сложнее деплой (тяжёлая библиотека)
* Меньше гибкости в тюнинге
* Моложе — меньше community ресурсов

* Быстрее всего на CPU - **LightGBM**
* Лучше с категориями - **CatBoost**
* Стабильность и гибкость - **XGBoost**
* Меньше тюнинга - **CatBoost**
* Huge scale (>10M) - **LightGBM** (GOSS) или **XGBoost** (distributed)

## Как выбирать (алгоритм для инженера)

```
1. Есть ли в данных категориальные признаки?
   ├─ Да, много (20%+), высокая кардинальность → CatBoost
   └─ Нет или мало (можно закодировать) → дальше

2. Какой объём данных?
   ├─ < 100K строк → XGBoost (надёжно, предсказуемо)
   ├─ 100K–1M → XGBoost или LightGBM (оба хороши)
   └─ > 1M строк → LightGBM (скорость и память)

3. Ключевое требование?
   ├─ Максимальная точность (соревнование) → LightGBM (после настройки)
   ├─ Быстрый инференс (real-time) → CatBoost
   ├─ Простота и стабильность → XGBoost
   └─ Минимум настроек → CatBoost
```

Начните с LightGBM для больших данных или CatBoost при обилии категорий. Если результат не устраивает — попробуйте другой. Часто качество близко, разница в скорости и удобстве.
Все три — отличные инструменты. Часто разница в 1-2% качества не стоит усложнения инфраструктуры. Выбирайте тот, который проще интегрировать в ваш текущий стек.

## Быстрый benchmark

```py
import time
from sklearn.datasets import make_classification
from sklearn.model_selection import train_test_split
from sklearn.metrics import roc_auc_score

# Генерация данных
X, y = make_classification(n_samples=100_000, n_features=50, 
                           n_informative=30, random_state=42)
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

models = {
    'XGBoost': xgb.XGBClassifier(n_estimators=100, tree_method='hist', n_jobs=-1),
    'LightGBM': lgb.LGBMClassifier(n_estimators=100, n_jobs=-1),
    'CatBoost': CatBoostClassifier(iterations=100, verbose=0, thread_count=-1)
}

results = {}
for name, model in models.items():
    start = time.time()
    model.fit(X_train, y_train)
    train_time = time.time() - start
    
    start = time.time()
    preds = model.predict_proba(X_test)[:, 1]
    pred_time = time.time() - start
    
    auc = roc_auc_score(y_test, preds)
    results[name] = {'AUC': auc, 'Train Time': train_time, 'Predict Time': pred_time}
    
    print(f"{name:12} | AUC: {auc:.4f} | Train: {train_time:.2f}s | Predict: {pred_time:.3f}s")

# Примерный вывод:
# XGBoost    | AUC: 0.8921 | Train: 12.45s | Predict: 0.234s
# LightGBM   | AUC: 0.8935 | Train: 2.18s  | Predict: 0.089s  ← 🚀
# CatBoost   | AUC: 0.8956 | Train: 28.67s | Predict: 0.312s  ← 🎯
```

## Правило 80/20:
