# Теорема Байеса

Теорема Байеса описывает, как обновлять вероятность гипотезы при получении новых данных. Она переворачивает условную вероятность: вместо того чтобы спрашивать «какова вероятность данных при условии гипотезы?», мы спрашиваем «какова вероятность гипотезы при условии наблюдаемых данных?».

* `P(A∣B)`  — **апостериорная вероятность** (posterior). Вероятность гипотезы A  после наблюдения данных B . Это то, что мы хотим найти.
* `P(B∣A)`  — **правдоподобие** (likelihood). Вероятность наблюдения данных B , если гипотеза A  верна.
* `P(A)`  — **априорная вероятность** (prior). Начальная вероятность гипотезы до получения данных.
* `P(B)`  — **полная вероятность данных** (marginal likelihood или evidence). Нормализующий множитель, который гарантирует, что сумма всех апостериорных вероятностей равна 1.

Формула:
```
P(A∣B) =  P(B∣A)*P(A) / P(B)

P(A∣B) = Правдоподобие * Априорная вероятность / полная вероятность данных
```

## Разбор:

* Болезнь встречается у 1% населения: `P(болен) = 0.01`
* Тест правильно выявляет болезнь у 95% больных: `P(тест+∣болен) = 0.95`
* Тест даёт ложноположительный результат у 5% здоровых: `P(тест+∣здоров) = 0.05`

Вопрос: Человек получил положительный тест. Какова вероятность, что он действительно болен?

Решение:

```
P(болен∣тест+) = 0.95*0.01 / 0.95*0.01 + 0.05*0.99 = 
= 0.0095 / 0.0095 + 0.0495 =
= 0.0095 / 0.059 ≈
≈ 0.161 
```

Ответ: `~16.1%` Даже при положительном тесте вероятность болезни невысока из-за низкой базовой частоты болезни и относительно высокого уровня ложноположительных результатов. Это часто неинтуитивно для людей (эффект базовой частоты).

**Даже с точным тестом из-за редкости болезни большинство положительных результатов будут ложными. Поэтому в ML мы всегда смотрим на Precision, а не только на Accuracy."**

**Accuracy** — это доля всех правильных ответов модели.

```
Accuracy = TP+TN / TP+TN+FP+FN
```

**Precision** — это доля реально больных среди всех, кому модель сказала «болен».

```
Precision = TP / TP+FP
```

* TP (True Positive) — правильно предсказали «болен»
* TN (True Negative) — правильно предсказали «здоров»
* FP (False Positive) — ошибочно сказали «болен» здоровому
* FN (False Negative) — ошибочно сказали «здоров» больному

`То есть теорема Байеса даёт вам формулу для расчёта Precision через известные параметры. Формула Байеса для медицинского теста — это ровно Precision`

**Precision** = ~16.1%
То есть только 16.1% из тех, кому тест сказал «болен», действительно больны.

А теперь посчитаем Accuracy на тех же цифрах (город 100 000 человек)

**Accuracy** = 95%
```
Accuracy = TP+TN / 100000 =
= 990+94050 / 100000 =
= 95040 / 100000 =
= 0.9504 = 95%
```

**Парадокс**: Accuracy высокая (95%), но модель бесполезна. Почему? Потому что она просто говорит «здоров» почти всем — и оказывается права в 95 случаев из 100 (ведь здоровых 99%).

`Accuracy бесполезна при сильном дисбалансе классов. В нашем примере Accuracy = 95%, но Precision = 16.7% — модель не умеет находить больных, она просто всегда предсказывает «здоров». Байесовский вывод показывает, что даже при «точном» тесте апостериорная вероятность болезни (она же Precision) низка из-за маленькой априорной вероятности и большого числа ложных срабатываний.`

### Пример из найма

У вас два рекрутера:
* Первый (опытный) верно оценивает кандидата с вероятностью 0.9, ошибается с 0.1.
* Второй (стажёр) делает это с вероятностями 0.6 и 0.4.
* Вероятность того, что случайный кандидат хороший 0.1

Кандидат получил положительную оценку. С какой вероятностью его оценивал опытный рекрутер?

```
P(A∣B) = 0.9×0.5 / 0.9×0.5 + 0.6×0.5 = 0.45 / 0.75 = 0.6
```

Это не вероятность того, что кандидат хороший. Это вероятность того, что положительную оценку дал опытный рекрутер (а не стажёр), если известно, что оценка положительная.

**Но если подчитать какова вероятность что кандидат хороший:**

Случай 1: Опытный рекрутер дал положительную оценку

```
P(хороший∣тест+ от опытного) = 0.9×0.1 / 0.9×0.1 + 0.1×0.9 =
= 0.09 / 0.09 + 0.09 =
= 0.09 / 0.18 =
= 0.5 
```
**Ответ: 50%**

Случай 2: Стажёр дал положительную оценку

```
P(хороший∣тест+ от стажёра) = 0.6×0.1 / 0.6×0.1 + 0.4×0.9 =
= 0.06 / 0.06 + 0.36 =
= 0.06 / 0.42 ≈
≈ 0.143 
```

**Ответ: ~14.3%**

Случай 3: Оба рекрутера дали положительную оценку (независимо)

```
P(оба+∣хороший) = 0.9×0.6 = 0.54
P(оба+∣плохой) = 0.1×0.4 = 0.04 

P(хороший∣оба+) = 0.54×0.1 / 0.54×0.1 + 0.04×0.9 =
= 0.054 / 0.054 + 0.036 = 
= 0.054 / 0.09 =
= 0.6 
```

**Ответ: 60%**

## Итог

Теорема Байеса описывает, как обновлять вероятность гипотезы при появлении данных. Формула: `P(A∣B) =  P(B∣A)*P(A) / P(B)`. В контексте ML она важна по трём причинам:
* Наивный байесовский классификатор строит это соотношение для каждого класса;
* Она объясняет, почему редкое событие трудно обнаружить даже точным тестом — из-за низкого априорного шанса;
* На ней основан байесовский вывод, используемый в байесовских нейросетях и Gaussian processes.

### Вопросы:

* **Почему `P(B)` — это сумма?** Потому что это полная вероятность события `B` через все возможности (`A` и не `A`).

* **"Наивный" наивный байесовский классификатор — что значит "наивный"?** Предположение о независимости признаков при условии класса. Обычно оно нарушено, но модель часто всё равно работает.
* **Если P(A) очень маленькая, что произойдёт?** Апостериорная вероятность также будет мала, даже при сильном свидетельстве — это ключевая интуиция, которую проверяют.

* **Как теорема Байеса связана с регуляризацией?** Максимизация апостериорной вероятности (MAP-оценка) = максимизация правдоподобия (MLE) + логарифм априорного распределения весов.
Если взять априорное распределение Лапласа — получится L1-регуляризация (Lasso). Если распределение Гаусса — L2 (Ridge).
Теорема Байеса напрямую даёт регуляризатор как способ внести наши доэкспериментальные знания о весах.
