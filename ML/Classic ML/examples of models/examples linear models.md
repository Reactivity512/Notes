# Пример LinearRegression

```py
import numpy as np
from sklearn.linear_model import LinearRegression

# Предсказываем цену квартиры: площадь (м²) и этаж
# X = [площадь, этаж]
X_train = np.array([
    [40, 5],   # 40 м², 5 этаж → цена 4 млн
    [60, 3],   # 60 м², 3 этаж → цена 5.5 млн
    [80, 10],  # 80 м², 10 этаж → цена 9 млн
])
y_train = np.array([4.0, 5.5, 9.0])

model = LinearRegression().fit(X_train, y_train)

print("coef_:", model.coef_)           # [0.09722222 0.22222222]
print("intercept_:", model.intercept_) # -0.9999999999999991

# coef_ показывает важность признаков — можно отфильтровать неважные
# Аномальные coef_ (огромные значения) → признак мультиколлинеарности или неотмасштабированные данные

# coef_[0] = 0.1 — каждый дополнительный м² добавляет 0.1 млн к цене
# coef_[1] = 0.3 — каждый этаж выше добавляет 0.3 млн
# intercept_ = 0.5 — базовая цена (если бы площадь и этаж были 0)

# Формула предсказания: цена=0.1×площадь+0.3×этаж+0.5
```

```py
# Автоматическое предсказание
model.predict([[50, 7]])  # array([5.41666667])
```

```py
coef = np.array([0.09722222, 0.22222222])
intercept = -0.9999999999999991

# Ручной расчёт предсказаний
predictions = X_train @ coef + intercept  # матричное умножение

print("Ошибка:", np.abs(predictions - y_train).max())  # должна быть ~0
# Ошибка: 1.999999987845058e-07
```

Почему intercept отрицательный?
intercept = -1.0 означает: Базовая цена квартиры с 0 м² на 0 этаже = -1 млн
Это математический артефакт — в реальности такой квартиры не существует, но модели нужен этот параметр для лучшей аппроксимации имеющихся данных.

```py
# Проверять размерность	coef_.shape должен совпадать с n_features
# n_features — это количество признаков (столбцов) в данных. (площадь, этаж = 2 признака)

```

## Пример схематически

### 1.Самый простой случай: 2D (один признак)

Представь, что мы предсказываем цену квартиры только по площади. Это одномерный мир.

```
Цена (тыс.$)
    |
500 |                                          /
    |                                       /
400 |                                    /
    |                                 /
300 |                              /
    |                           /
200 |                        /
    |                     /
100 |                  /
    |               /
   0 +---------------------------------------------------> Площадь (м²)
     0    20    40    60    80   100   120   140
```

* Модель провела прямую линию через точки (данные о продажах).
* Формула: Цена = 2.5 * Площадь + 50 (примерно)
* Чтобы предсказать цену для квартиры 80 м² → проводим вертикаль вверх до линии → получаем ~250 тыс.

**Это просто prediction = weight * feature + bias.**

### 2.Два признака: плоскость (3D-пространство)

Теперь у нас площадь и количество комнат. Модель строит уже не линию, а плоскость.

```
Цена
  ↑
  |                    ╱
  |                  ╱
  |                ╱
  |              ╱
  |            ╱
  |          ╱
  |        ╱
  |      ╱
  |    ╱
  |  ╱
  |╱
  +------------------------→ Площадь
  ╲
   ╲
    ╲
     ╲
      ╲ (комнаты уходят вглубь)
```

* Ось X — площадь
* Ось Z (вглубь) — количество комнат
* Ось Y — цена

**Цена = w₁ * Площадь + w₂ * Комнаты + bias**

Теперь признака два, но сложность та же O(n) — просто два умножения и сложение.

### 3.Бинарная классификация: разделяющая прямая

Здесь модель не предсказывает число, а разделяет два класса (например, "одобрено кредит" vs "отказано").

```
Доход
  ↑
  |                    ○ ○ ○ ○
  |                 ○ ○ ○ ○
  |              ○ ○ ○ ○
  |           ○ ○ ○ ○
  |        ○ ○ ○ ○
  |     × × × ×
  |  × × × ×
  | × × ×
  |××
  +--------------------------------→ Возраст
```

* ○ (кружочки) — одобренные клиенты
* × (крестики) — отказанные

Прямая линия разделяет их. Всё, что выше прямой → одобрено, ниже → отказано.

**w₁ * Возраст + w₂ * Доход + bias = 0**
* `> 0` → класс 1 (одобрено)
* `< 0` → класс 0 (отказано)

### 4.Три признака: гиперплоскость

Когда признаков 3 (возраст, доход, кредитная история), модель строит плоскость в 3D, которая делит куб на две части.

Когда признаков 4 и больше, это называется гиперплоскость — тот же принцип, но в многомерном пространстве. Наш мозг не может это визуализировать, но компьютер отлично работает:
```
Для 100 признаков:
prediction = w₁*x₁ + w₂*x₂ + ... + w₁₀₀*x₁₀₀ + bias
```

Алгоритм обучения просто находит такие веса (w), чтобы:
* Для регрессии: прямая проходила как можно ближе ко всем точкам.
* Для классификации: прямая разделяла классы с максимальным зазором.

Представь, что у нас один признак (x) и вес (w):

* `w = 0` Горизонтальная прямая (предсказание всегда равно bias)
* `w > 0` Прямая идёт вверх (чем больше x, тем больше предсказание)
* `w < 0` Прямая идёт вниз (чем больше x, тем меньше предсказание)
* `|w| большое` Прямая крутая (признак очень влияет на результат)
* `|w| маленькое` Прямая пологая (признак слабо влияет)


# Пример Ridge

Ridge-регрессия — один из самых практичных инструментов для борьбы с переобучением и мультиколлинеарностью. 

Мультиколлинеарность — это ситуация, когда признаки сильно коррелируют друг с другом (линейно зависимы).

Аналогия:
* Площадь в м² - ✓ Независимый признак
* Площадь в футах - ✗ Тоже самое, просто в других единицах
* Количество комнат - ⚠ Часто коррелирует с площадью

Два признака рассказывают одну и ту же историю — это мультиколлинеарность.

Почему это проблема?
```py
import numpy as np
from sklearn.linear_model import LinearRegression

# Данные: X1 и X2 почти идентичны (мультиколлинеарность)
X = np.array([
    [10, 10.1],   # почти одинаковые значения
    [20, 20.1],
    [30, 30.2],
    [40, 40.1],
])
y = np.array([15, 25, 35, 45])

model = LinearRegression().fit(X, y)
print("coef_:", model.coef_)  # [100.0, -90.0] — огромные веса с разными знаками!
print("intercept_:", model.intercept_)
```

**Модель может использовать `100*X1 - 90*X2` или `10*X1 - 0*X2` — результат одинаковый, но веса совершенно разные. При небольшом изменении данных веса скачут wildly.**

Если площадь и комнаты коррелируют на 95%:
* Модель даёт вес 50 площади и 0 комнатам
* Или 0 площади и 50 комнатам  
* Или 25 и 25
→ Непонятно, что реально влияет на цену!

Как решать

| Способ |Когда использовать | Код
|--|--|--
| Удалить один из признаков | Один признак явно дублирует другой| `df.drop('area_sqft', axis=1)`
| PCA (Principal Component Analysis) | Много коррелирующих признаков, нужно сократить размерность | `PCA(n_components=0.95)`
| Регуляризация (Ridge/Lasso) | Сохранить все признаки, но стабилизировать веса	| `Ridge(alpha=1.0)`
| Объединить признаки | Признаки логически связаны (площадь + комнаты → площадь_на_комнату) | Создать новый признак

### Пример решения через Ridge

```py
from sklearn.linear_model import Ridge
from sklearn.preprocessing import StandardScaler

# Ridge добавляет L2-штраф — веса становятся маленькими и стабильными
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

model = Ridge(alpha=10.0).fit(X_scaled, y)
print("Ridge coef:", model.coef_)  # [4.95, 4.95] — стабильные, одинаковые веса
```

Если сравнивать в примером выше то нужно: просто заменить `LinearRegression` на `Ridge` и добавь параметр `alpha` (сила регуляризации):
```py
import numpy as np
from sklearn.linear_model import Ridge  # ← замена

# Данные те же
X_train = np.array([
    [40, 5],
    [60, 3],
    [80, 10],
])
y_train = np.array([4.0, 5.5, 9.0])

# Ridge вместо LinearRegression
model = Ridge(alpha=1.0).fit(X_train, y_train)  # ← добавлен alpha

print("coef_:", model.coef_)
print("intercept_:", model.intercept_)
```
Но с такими данными разница минимальна — мультиколлинеарности нет, признаки не коррелируют.

**Когда Ridge реально помогает**
**Всегда используй `StandardScaler` перед регуляризацией, если признаки в разных масштабах!**

```py
import numpy as np
from sklearn.linear_model import Ridge
from sklearn.preprocessing import StandardScaler

# Предсказываем цену квартиры: площадь (м²) и этаж
# X = [площадь, этаж]
X_train = np.array([
    [40, 5, 430.4],   # 40 м², 5 этаж, 430.4 sqft → цена 4 млн
    [60, 3, 645.6],   # 60 м², 3 этаж, 645.6 sqft → цена 5.5 млн
    [80, 10, 861.0],  # 80 м², 10 этаж, 861.0 sqft → цена 9 млн
])
y_train = np.array([4.0, 5.5, 9.0])

# 1. Масштабируем признаки (среднее=0, std=1)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

model = Ridge(alpha=10.0).fit(X_scaled, y_train)

print("coef_:", model.coef_)           # [0.34440627 0.29477143 0.34442709]
print("intercept_:", model.intercept_)
```

Без `StandardScaler` Веса будут [0.00067202 0.1275668  0.01006723]
```
Площадь м²:     40  →  вес 0.00067  (множитель ~60)
Этаж:           5   →  вес 0.128    (множитель ~1)
Площадь фут²:   430 →  вес 0.01     (множитель ~43)
```

Чтобы сравнить реальное влияние, нужно умножить вес на типичное значение признака:
|Признак | Вес × Среднее значение | Реальный вклад
|--|--|--
| Площадь м²| 0.00067 × 60 | 0.04 млн
| Этаж | 0.128 × 6 | 0.77 млн ← доминирует!
| Площадь фут² | 0.01 × 645 | 6.45 млн ← перекрутило

Ridge штрафует квадраты весов. Чтобы минимизировать штраф, модель делает веса пропорционально обратно масштабу признака:
* Фут² имеет большие значения (430-861) → вес маленький (0.01)
* М² имеет средние значения (40-80) → вес ещё меньше (0.00067)

Но фактически фут² и м² — одно и то же! Модель распределила влияние между ними неправильно из-за разных единиц измерения. Необходимо масштабирование перед Ridge.


### Ключевые моменты для ML-инженера: 

| Сценарий | Действие
|--|--
| Линейная регрессия | Обязательно проверять VIF, использовать Ridge
| Деревья/Random Forest | Не чувствительны к мультиколлинеарности — можно игнорировать
| Нейросети | Менее чувствительны, но градиенты могут быть нестабильны
| Feature selection | Удаляй признаки с высоким VIF перед обучением

**Правило: Мультиколлинеарность критична для интерпретируемости и линейных моделей, но менее важна для деревьев и ансамблей.**

# Пример Lasso

Lasso — отличается от Ridge тем, что может обнулять веса (отбор признаков)

Пример:
```py
import numpy as np
from sklearn.linear_model import Lasso
from sklearn.preprocessing import StandardScaler

# Данные: 4 признака, но только 2 реально влияют на цену
# X = [площадь_м2, этаж, расстояние_до_метро_км, цвет_стен_код]
X_train = np.array([
    [40, 5, 2.5, 1],   # цена 4 млн
    [60, 3, 1.0, 2],   # цена 5.5 млн
    [80, 10, 0.5, 1],  # цена 9 млн
    [50, 7, 3.0, 3],   # цена 5 млн
    [70, 2, 1.5, 2],   # цена 6.5 млн
])
y_train = np.array([4.0, 5.5, 9.0, 5.0, 6.5])

# Масштабируем (Lasso чувствителен к масштабу!)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X_train)

# Lasso с разными значениями alpha
for alpha in [0.01, 0.1, 1.0, 10.0]:
    model = Lasso(alpha=alpha).fit(X_scaled, y_train)
    print(f"\nalpha={alpha}:")
    print(f"  coef_: {model.coef_}")
    print(f"  ненулевых признаков: {np.sum(model.coef_ != 0)}")
```

Результат:
```
alpha=0.01:
  coef_: [2.45, 0.89, -0.12, 0.0]
  ненулевых признаков: 3

alpha=0.1:
  coef_: [2.38, 0.76, 0.0, 0.0]
  ненулевых признаков: 2

alpha=1.0:
  coef_: [1.95, 0.0, 0.0, 0.0]
  ненулевых признаков: 1

alpha=10.0:
  coef_: [0.0, 0.0, 0.0, 0.0]
  ненулевых признаков: 0  ← все обнулилось!
```

**Что происходит**
| alpha | Эффект
|--|--
| Маленький (0.01) | Почти как `LinearRegression` — сохраняет все признаки
| Средний (0.1) | Обнуляет слабые признаки (расстояние до метро, цвет стен)
| Большой (1.0) | Оставляет только самый важный (площадь)
| Очень большой (10.0) | Обнуляет всё — слишком сильный штраф

## Сравнение Lasso vs Ridge

```py
from sklearn.linear_model import Ridge, Lasso

alpha = 0.1

# Ridge — веса малые, но ненулевые
ridge = Ridge(alpha=alpha).fit(X_scaled, y_train)
print("Ridge:", ridge.coef_)  # [2.1, 0.5, -0.3, 0.1] — все признаки остались

# Lasso — некоторые веса точно 0
lasso = Lasso(alpha=alpha).fit(X_scaled, y_train)
print("Lasso:", lasso.coef_)  # [2.38, 0.76, 0.0, 0.0] — отбор признаков!
```

## Итог

| Задача | Метод
|--|--
| Просто стабилизировать веса | Ridge
| Уменьшить количество признаков | Lasso
| И то, и другое | ElasticNet (комбинация L1 + L2)

**Lasso = автоматический feature selection — удобно, когда признаков много и нужно упростить модель.**


# Пример LogisticRegression 

LogisticRegression — это классификатор для бинарной классификации (да/нет, 0/1, спам/не спам). Несмотря на название "регрессия", это не регрессия, а классификация.

С точки зрения ML-инженера, это инструмент для быстрых, интерпретируемых и надежных решений.

Вы уже знаете, что линейная модель считает `w₁*x₁ + ... + bias` (как в линейной регрессии). Проблема в том, что это число может быть от `-∞` до `+∞`, а нам нужна вероятность от 0 до 1.

Решение: Пропустить это число через сигмоиду (S-образную функцию), которая "сожмет" его в отрезок [0, 1].

Вот как это выглядит:
* Линейная часть (как у регрессии): `z = w₁*x₁ + w₂*x₂ + ... + bias`

* Сигмоида (превращение в вероятность): Применяем логистическую функцию.
`p = 1 / (1 + e^(-z))`

Результат: p — это вероятность того, что объект относится к классу 1 (например, "одобрено"). Если p > 0.5, предсказываем класс 1, иначе — класс 0.

```py
import numpy as np
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler

# Данные: площадь и этаж → покупают (1) или нет (0)
X = np.array([
    [40, 5],   # не купили
    [60, 3],   # не купили  
    [80, 10],  # купили
    [100, 15], # купили
    [45, 8],   # не купили
    [90, 12],  # купили
])
y = np.array([0, 0, 1, 1, 0, 1])

# Масштабирование важно!
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Обучение
model = LogisticRegression().fit(X_scaled, y)

print("coef_:", model.coef_)        # веса признаков
print("intercept_:", model.intercept_)  # смещение

# Предсказание класса
print("Классы:", model.predict(X_scaled))  # [0, 0, 1, 1, 0, 1]

# Предсказание вероятностей
proba = model.predict_proba(X_scaled)
print("Вероятности класса 1:\n", proba[:, 1])
# [0.13, 0.19, 0.67, 0.92, 0.24, 0.82]
```

| Параметр | Значение | Когда менять
|--|--|--
| `C` | Обратная сила регуляризации (default=1.0) | Меньше C = сильнее штраф, проще модель
| `penalty` | Тип регуляризации: 'l2', 'l1', 'elasticnet' | L1 для отбора признаков
| `solver` | Алгоритм оптимизации | 'liblinear' для L1, 'lbfgs' для L2
| `max_iter` | Максимум итераций | Увеличь, если warning о сходимости

```py
# L1-регуляризация (отбор признаков)
model_l1 = LogisticRegression(penalty='l1', solver='liblinear', C=0.1)

# Сильная регуляризация (простая модель)
model_strong = LogisticRegression(C=0.01)

# Слабая регуляризация (сложная модель, риск переобучения)
model_weak = LogisticRegression(C=100)
```

### Инженерный взгляд: что важно знать

| | LinearRegression | LogisticRegression
|--|--|--
| Задача | Регрессия (число) | Классификация (класс)
| Выход | Любое число (`-∞` до `+∞`) | Вероятность (0 до 1)
| Функция потерь | MSE | Log Loss (кросс-энтропия)

* **MSE** — если в данных есть выбросы (например, аномальные цены), модель будет "дергаться", чтобы их учесть. Нужно либо чистить данные, либо использовать MAE или Huber.
* **Log Loss** — если модель выдает вероятности 0.999 или 0.001, но ошибается — получаешь inf. Значит, нужна clipping (обрезать вероятности в [0.01, 0.99]).
* Градиенты — обе функции гладкие (хорошо для градиентного спуска), но Log Loss может долго сходиться, если классы сильно несбалансированы (нужен class_weight='balanced').

**MSE (Mean Squared Error)** штрафует квадрат ошибки. Большие ошибки → очень большой штраф. Чувствительна к выбросам.

**Log Loss (Binary Cross-Entropy)** штрафует за уверенно неверные предсказания. Если модель сказала "99%", а ошиблась — штраф огромный.

**Важные аспекты:**

1. Выход: вероятность, а не просто класс

Плохая практика: использовать только `model.predict()` (вернет 0 или 1). Хорошая практика: использовать `model.predict_proba()`.

```py
# ❌ Плохо: просто "одобрено" или "отказано"
is_approved = model.predict([age, income])

# ✅ Хорошо: вероятность одобрения (например, 0.87)
probability = model.predict_proba([age, income])[0, 1]
```

* Можно установить разные пороги (для рискованных кредитов — > 0.7, для надежных — > 0.3).
* Бизнес может ранжировать клиентов по "скору".

2. Веса модели и интерпретация

Логистическая регрессия дает веса, которые показывают влияние признаков на логарифм шансов. На практике проще смотреть на `exp(weight)`.
```py
coef = model.coef_[0]
print(f"Шанс одобрения растет в {np.exp(coef[0]):.2f} раз при увеличении дохода")
```

```
Признак "доход": exp(вес) = 1.85
Это значит: при увеличении дохода на 1 единицу (после масштабирования), 
шансы получить кредит вырастают на 85%.
```

3. Параметр `C` — регуляризация

В отличие от `alpha` в линейной регрессии, в `LogisticRegression` используется параметр `C`, который является обратным к силе регуляризации.

* Маленький `C` (0.01): Сильная регуляризация. Простая модель, которая может недообучиться.
* Большой `C` (100): Слабая регуляризация. Сложная модель, которая может переобучиться.
* `C` = 1.0: Значение по умолчанию, хорошая отправная точка.

```py
# L2 регуляризация (по умолчанию) с C = 0.1
model = LogisticRegression(C=0.1, penalty='l2')
```

Правило для инженера:
* `penalty='l2'` (Ridge) — используйте по умолчанию.
* `penalty='l1'` (Lasso) — используйте, если нужно отобрать признаки (много лишних колонок).

4. Solvers — не магия, а выбор алгоритма

В зависимости от размера данных, нужно выбирать правильный "движок" для обучения.

| Solver | Когда использовать | Плюсы/минусы
|--|--|--
| `lbfgs` | По умолчанию (данные до 10К строк) | Стабильный, стандарт.
| `liblinear` | Небольшие данные, L1-регуляризация | Единственный, кто поддерживает L1.
| `saga` | Большие данные (>100К строк), L1/L2/ElasticNet | Быстрый, гибкий, поддерживает все виды регуляризации.

```py
# Для большого датасета с L1-регуляризацией
model = LogisticRegression(penalty='l1', solver='saga', C=0.5, max_iter=1000)
```

### Полный пример

```py
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.linear_model import LogisticRegression

# 1. Препроцессинг
preprocessor = ColumnTransformer([
    ('num', StandardScaler(), ['age', 'income']), # Обязательно масштабируем!
    ('cat', OneHotEncoder(handle_unknown='ignore'), ['city']) # Категории
])

# 2. Модель (логистическая регрессия)
model = Pipeline([
    ('preprocessor', preprocessor),
    ('classifier', LogisticRegression(
        C=1.0,                # Регуляризация
        solver='lbfgs',       # Стандартный солвер
        max_iter=1000,        # Достаточно итераций
        class_weight='balanced' # Важно для несбалансированных классов!
    ))
])

# 3. Обучение
model.fit(X_train, y_train)

# 4. В продакшене (FastAPI)
probability = model.predict_proba(X_new)[0, 1]  # Берем вероятность класса 1
```

##  Когда логистическая регрессия — плохой выбор?

1. Несбалансированные классы: Вы работаете с `class_weight='balanced'`, но при сильном дисбалансе (например, 1:1000) лучше использовать специальные методы (апсемплинг, другие метрики).

2. Сложные нелинейные зависимости: Если признаки взаимодействуют сложным образом (например, "работает только для женщин старше 40"), деревья или бустинг справятся лучше без ручного создания новых признаков.

3. Очень много признаков (миллионы): `LogisticRegression` со `solver='saga'` справится, но для разреженных данных может быть эффективнее использовать `SGDClassifier` с `loss='log_loss'`.

## LogisticRegression vs. Другие модели

| Модель | Скорость | Качество (по умолчанию) | Интерпретируемость | Когда выбрать
|--|--|--|--|--
| LogisticRegression | Очень быстро | Хорошее | Отличная | Базовый выбор для классификации.
| Decision Tree | Быстро | Среднее (склонно к переобучению) | Хорошая | Когда важна интерпретируемость, но есть нелинейности.
| Random Forest | Средне | Очень хорошее | Плохая | Когда нужно высокое качество, а скорость не критична.
| XGBoost | Медленно (обучение) | Лучшее | Плохая | Когда вам нужно выжать максимум качества для соревнования или задачи.
