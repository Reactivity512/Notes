# SVM и KNN

SVM и KNN. Это два классических алгоритма, которые нужно знать, но в современном классическом ML (на табличных данных) они используются редко.

**Главная причина**: для продакшена они либо медленные, либо сложные в масштабировании, либо уступают по качеству градиентному бустингу (XGBoost/LightGBM) и случайному лесу.

# KNN (K-Nearest Neighbors) — Ленивый классификатор

Принцип работы: `"Скажи мне, кто твой друг, и я скажу, кто ты".`

Чтобы классифицировать новый объект, KNN:
* Находит K самых близких соседей в обучающей выборке.
* Голосует: какой класс встречается чаще, тот и предсказываем.

KNN — алгоритм "ленивого" обучения: не строит модель, а запоминает все данные и ищет похожие примеры при предсказании.

```py
from sklearn.neighbors import KNeighborsClassifier

knn = KNeighborsClassifier(n_neighbors=5)  # K=5
knn.fit(X_train, y_train)  # Запоминает все данные (не обучается!)
pred = knn.predict(X_new)   # Считает расстояния до всех точек в памяти
```

Принцип работы:
* Обучение: просто сохранить все (X, y) в память
* Предсказание: найти K ближайших соседей к новой точке → усреднить их ответы

## Для инженера: почему KNN почти не используют в продакшене

| Проблема | Почему плохо | Что делать
|--|--|--
| Медленный инференс | Каждый запрос требует сравнения со ВСЕМИ тренировочными данными. 1 млн точек → 1 млн вычислений расстояний. | Использовать приближенные методы (ANN) или не использовать KNN
| Проклятие размерности | При >20 признаках расстояния становятся бессмысленными (все точки одинаково далеки). | Уменьшать размерность (PCA)
| Требует масштабирования | Признак "доход" (0-1 млн) перевешивает "возраст" (0-100). | Обязательно StandardScaler
| Хранит все данные | Модель = вся тренировочная выборка. 1 млн строк * 100 признаков = 400 MB. | Использовать только для маленьких данных

## Когда KNN всё же используют

* Рекомендательные системы ("похожие товары")
* Медицина (поиск похожих пациентов)
* Аномалии (если у точки нет близких соседей)
* Маленькие данные (<10K строк, <20 признаков)

```py
# Единственный production-сценарий
knn = KNeighborsClassifier(n_neighbors=5, algorithm='ball_tree')
knn.fit(X_small, y_small)  # X_small - не больше 10-50K строк
```

### Пример в коде:

```py
import numpy as np
from sklearn.neighbors import KNeighborsClassifier, KNeighborsRegressor
from sklearn.preprocessing import StandardScaler

# Классификация: предсказать тип квартиры (эконом/комфорт/люкс)
X = np.array([
    [40, 5],   # 40 м², 5 этаж — эконом
    [45, 3],   # эконом
    [80, 10],  # комфорт
    [85, 12],  # комфорт
    [150, 20], # люкс
    [160, 25], # люкс
])
y = np.array(['эконом', 'эконом', 'комфорт', 'комфорт', 'люкс', 'люкс'])

# Масштабирование критично! (иначе этажи не заметятся на фоне метров)
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# K=3: смотрим на 3 ближайших соседа
knn = KNeighborsClassifier(n_neighbors=3)
knn.fit(X_scaled, y)

# Новая квартира: 50 м², 6 этаж
X_new = np.array([[50, 6]])
X_new_scaled = scaler.transform(X_new)

prediction = knn.predict(X_new_scaled)          # ['эконом']
probabilities = knn.predict_proba(X_new_scaled) # [0.67, 0.33, 0.0]

print(f"Класс: {prediction[0]}")
print(f"Вероятности: эконом={probabilities[0][0]:.2f}, комфорт={probabilities[0][1]:.2f}")
```

```
Класс: эконом
Вероятности: эконом=0.33, комфорт=0.00
```

**Регрессия с KNN:**

```py
# Предсказание цены по соседям
y_price = np.array([3.5, 3.8, 6.5, 7.0, 15.0, 16.5])  # цены в млн

knn_reg = KNeighborsRegressor(n_neighbors=2)
knn_reg.fit(X_scaled, y_price)

# Цена = средняя цена 2 ближайших соседей
price_pred = knn_reg.predict(X_new_scaled)
print(f"Предсказанная цена: {price_pred[0]:.2f} млн")  # ~3.65 (среднее 3.5 и 3.8)
```

```
Предсказанная цена: 3.65 млн
```

### Ключевые параметры

| Параметр | Что делает | Типичное значение
|--|--|--
| `n_neighbors` (K) | Сколько соседей учитывать | 	3-5 для малого датасета, 20-50 для большого
| `weights` | Как взвешивать соседей | `'uniform'` (равно) или `'distance'` (ближе = важнее)
| `metric` | Метрика расстояния | `'minkowski'` (p=2 = евклидово), `'manhattan'`
| `algorithm` | Как искать соседей | `'auto'`, `'ball_tree'`, `'kd_tree'`, `'brute'`

## Вывод для инженера

KNN — скорее инструмент для:
* Baseline ("а что если просто запомнить?")
* Рекомендаций (collaborative filtering)
* Поиска похожих (similarity search)

В production ML чаще:
* Approximate Nearest Neighbors (ANN): faiss, annoy, hnswlib
* Embeddings + cosine similarity в векторной БД (Pinecone, Weaviate)


# SVM (Support Vector Machine) — Мощный, но капризный

SVM ищет разделяющую гиперплоскость с максимальным зазором (margin) между классами.

Ключевая идея: не просто разделить классы, а сделать это с максимальным "отступом" от границы. Точки на границе — опорные векторы (support vectors).

```
Класс A: ○ ○ ○ ○
          \  |  /
           \ | /   <-- Margin (зазор)
            \|/
-------------+-------------  ← Разделяющая гиперплоскость
            /|\
           / | \
          /  |  \
Класс B: × × × ×
```

```
Вместо "просто разделить классы" → "разделить так, чтобы зазор был максимальным"

●——— margin ———●
   ○ ○    ● ●
  ○   ○  ●   ●
 ○     ○●     ●
  ○   ○  ●   ●
   ○ ○    ● ●
```

Опорные векторы — точки на границе margin, которые "держат" разделяющую линию. Остальные точки не влияют на модель.

## Трюк с ядром (Kernel Trick) — ключевая фича SVM

SVM может работать в бесконечномерном пространстве, не вычисляя координаты явно. Через ядра (kernels):

| Ядро | Формула | Когда использовать
|--|--|--
| **linear** | `x·y` | Линейно разделимые данные
| **poly** | `(γ·x·y + r)^d` | Нелинейные, но не очень сложные
| **rbf** (Radial Basis Function) | `exp(-γ\|\|x-y\|\|²)` | По умолчанию. Сложные нелинейности

```py
from sklearn.svm import SVC

# Линейный SVM (быстрый)
svm_linear = SVC(kernel='linear', C=1.0)

# RBF SVM (мощный, но медленный)
svm_rbf = SVC(kernel='rbf', C=1.0, gamma='scale')
```

Пример:
```py
import numpy as np
from sklearn.svm import SVC, SVR
from sklearn.preprocessing import StandardScaler

# Классификация: разделить два класса
X = np.array([
    [1, 2], [2, 3], [2, 1],  # класс 0
    [6, 5], [7, 7], [8, 6],  # класс 1
])
y = np.array([0, 0, 0, 1, 1, 1])

# Масштабирование обязательно!
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Linear SVM
svm_linear = SVC(kernel='linear', C=1.0)
svm_linear.fit(X_scaled, y)

# Предсказание
X_new = np.array([[3, 2], [7, 6]])
X_new_scaled = scaler.transform(X_new)
print(f"Предсказания: {svm_linear.predict(X_new_scaled)}")  # [0, 1]

# Опорные векторы
print(f"Опорных векторов: {len(svm_linear.support_)}")  # 2-3 точки
print(f"Индексы: {svm_linear.support_}")
```

```
Предсказания: [0 1]
Опорных векторов: 2
Индексы: [1 3]
```

## Для инженера: почему SVM редко в продакшене

| Проблема | Почему плохо | Что делать
|--|--|--
| Медленное обучение | O(n²) до O(n³). 100K точек → часы/дни. | Использовать линейное ядро или SGDClassifier
| Медленный инференс | Зависит от числа опорных векторов (может быть много). | Использовать линейное ядро
| Плохо с большими данными | >50K строк уже проблема. | Переходить на бустинг или нейросети
| Чувствителен к масштабу | Обязательно масштабировать! | StandardScaler
| Сложная настройка | `C` (регуляризация), `gamma` (для RBF). | GridSearch

**Параметры, которые нужно тюнить:**

| Параметр | Что делает | Эффект
|--|--|--
| `C` | Регуляризация | Меньше C → шире margin, больше ошибок на train, лучше generalization
| `gamma` (для RBF) | "Радиус" влияния точки | Больше gamma → сложнее граница, риск переобучения
| `kernel` | Тип разделяющей поверхности | linear для скорости, rbf для точности

### Когда SVM всё же используют

* Классификация текстов (линейный SVM с TF-IDF) — до сих пор конкурентен
* Распознавание изображений (в паре с HOG-фичами) — до эры нейросетей
* Маленькие выборки (<10K) с большим числом признаков
* Когда важна интерпретируемость (линейное ядро даёт веса)

### Когда НЕ использовать SVM

| Проблема | Почему | Альтернатива
|--|--|--
| Большие датасеты (>100k) | Слишком медленно | Logistic Regression, LinearSVC
| Нужна вероятность | Дорого считать | Logistic Regression
| Много классов | One-vs-One/OvR сложно | Random Forest, XGBoost
| Нужен online learning | Не поддерживает | SGDClassifier
| Интерпретация важна | Ядра — чёрный ящик | Linear models

## Сравнение KNN и SVM (инженерный взгляд)

| Характеристика | KNN	SVM (RBF) | SVM (linear)
|--|--|--
| Обучение | O(1) (просто запомнить) | O(n²) - O(n³) | O(n) - O(n²)
| Инференс | O(n) (медленно) | O(k) (k=опорные векторы) | O(k) (быстро)
| Память | Хранит все данные | Хранит опорные векторы | Хранит веса
| Масштабирование | Обязательно | Обязательно | Обязательно
| Высокая размерность | Плохо | Нормально (RBF) | Хорошо
| Нелинейности | Нет (только метрика) | Да (RBF/poly) | Нет
| Когда использовать | <50K строк, <20 признаков | <50K строк, качество важно | Тексты, разреженные данные

## Что происходит в реальном продакшене

**KNN**
* Почти не используется для классификации табличных данных.
* Живет в поиске похожих (FAISS, Annoy, HNSW) — приближенные методы для миллиардов векторов.

*SVM*
* LinearSVM (или `SGDClassifier(loss='hinge')`) используется в:
    - Классификации текстов (спам/не спам, тематики)
    - CTR-предсказании (реклама) — как бейзлайн
* SVM с RBF — редко, уступает XGBoost по качеству и скорости.

Что вместо них?
* Классификация табличных данных - XGBoost / LightGBM / CatBoost
* Поиск похожих (KNN) - FAISS, HNSW (приближенный поиск)
* Классификация текстов - LogisticRegression (с TF-IDF) или BERT
* Маленькие нелинейные данные - Random Forest (проще настройки)

## Итог

### KNN

**Плюсы:** простой, не обучается, хорошо для маленьких данных
**Минусы:** медленный инференс O(n), хранит все данные, плох с высокой размерностью
**Гиперпараметры:** `K` (число соседей), метрика расстояния (обычно евклидова)

### SVM
**Плюсы:** работает в высоких размерностях, ядровой трюк, теоретически обоснован
**Минусы:** медленное обучение, сложная настройка, плох с большими данными
**Гиперпараметры:** C (штраф за ошибки), gamma (для RBF), выбор ядра

## Почему не SVM/KNN?

На табличных данных с >50K строк XGBoost/LightGBM дают лучшее качество при меньшем времени обучения и инференса. KNN и SVM использую только для маленьких выборок или специфичных задач (тексты, поиск похожих).
